Overview

This is the Web interface of my Distributed Computing Project (COMP90019) in University of Melbourne in Semester 2 2014.

The project topic is: Exploring Melbourne’s Transport System through Twitter

Author: Yikai Gong -- yikaig@student.unimelb.edu.au


Goal:

  • Visualize public transport status based on social media (Twitter) data.

    • Can we identify congestion across the main roads of Melbourne in real-time?
    • What time, day, route is the best way to commute?

Data Resources:

  • Twitter – “targeted” real time harvesting via the Twitter API

  • Public Sector Mapping Agency (PSMA)

    • The definitive geospatial data provider for Australia includes all new roads, new houses etc
    • PSMA data made available through AURIN (www.aurin.org.au)

Services:

  • Targeted twitter harvesters (capturing tweets along main roads)

  • Web services for transport related tweets, clustering, sentiment analysis, …

  • Front-end visualisation services.


System Structure



Collected Data

  • Tweets in Melbourne: 7259004 records (From 22 September to 20 October)

  • Tweets on streets (in CBD and some main highway): 25753 records (From 22 September to 20 October)


Cloud Instance deployment

  • The real time tweets harvester and a tomcat server is deployed in the one of my instances in NeCTAR Research Cloud

  • Two streets harvesters and the couchDB database are deployed in another instance.

  • The harvester for collecting tweets in Melbourne is deployed in a third instance.


Help Info

  • This Web service now provides two analysis views. One is for looking at a specific date. Another is for looking at the days of week.

  • The Real time service is based on Web-socket (Check the browser support). By default, it will highlight tweets that are talking about traffic. User can also define some keywords to filter the tweets through a input form.
